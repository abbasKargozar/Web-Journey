HTML Hyperlinks: A Simple Yet Essential Tool

Creating hyperlinks in HTML is a fundamentally simple concept, yet it remains one of the most vital and frequently utilised features in web development. Hyperlinks are the glue that holds the web together, allowing users to navigate seamlessly between different pages and resources. 2.1 All in one

Sometimes you may want all of your external links to open in the same browser tab instead of creating a new tab every time. This can provide a cleaner browsing experience and prevent users from ending up with dozens of open tabs. To achieve this, assign the same custom name to the "target" attribute for every link, such as target = "NewTab". The first click creates a new tab with that name, and every subsequent click loads the new page into the same tab instead of opening another one. This behavior works with any custom target name, not just "NewTab". Keep in mind that if you use rel = "noopener" or rel = "noreferrer" with these links, some browsers and websites may no longer allow the named tab to be reused, so this technique may not work as expected. 


            <ul>

                <li><a href="https://github.com/abbasKargozar" target = "NewTab">open my github in new tab</a></li>
                <li><a href="https://example.com" target = "NewTab">open example.com in the same tab</a></li>
                <li><a href="https://www.w3schools.com" target = "NewTab">open W3Schools in the same tab</a></li>

            </ul>

2.2 Inline Frame

It may seem a little unusual to introduce inline frames <iframe> at this point, but there is a good reason for it. While working with the "target" attribute, you will likely come across reserved values such as "_self", "_parent", and "_top". These values are closely related to frames and iframes, so it is helpful to understand what they mean before using them. An <iframe> allows you to embed another webpage inside your own, creating a separate browsing context within the page. A common example is an embedded Google Map on a website—you are viewing an actual webpage displayed inside another webpage, as if a small browser window were built directly into the site. Understanding how iframes work will make the different "target" values much easier to understand.

As an example, **OpenStreetMap** is one of the most popular open-source mapping platforms and is widely used to embed interactive maps into websites. In this lesson, we use it to display a map inside an "<iframe>". Whenever you embed content from another source, make sure you respect the provider's license, terms of use, and copyright requirements. Always verify that you have permission to display the content and provide proper attribution when it is required. 


        <iframe width="550" height="400" src="https://www.openstreetmap.org/export/embed.html?bbox=...></iframe>



View Larger Map

It's also worth remembering that an "<iframe>" doesn't have to display an external website. You can use it to load different pages from your own website as well, which can be useful for dashboards, documentation, previews, or other isolated sections of an application. Later in this course, we'll build a few practical examples that demonstrate these use cases. We'll also revisit iframes in the Web Security section, where you'll learn about common iframe-related vulnerabilities and how modern browsers help protect against them.

3. Navigating Within the Same Page (Anchor Links)

For lengthy documents or single-page websites, jumping to a specific section is highly convenient. You can link directly to a specific element by appending a hash (#) followed by the element’s id to the href value, enabling smooth internal navigation.

A common example of this feature is the **"Back to Top"** button found on many websites. When clicked, it takes the user from the footer or any other part of the page back to the top by linking to an element with "#". for example I used: 

   <a href="#"> ^Go to top ^ </a>

Note: A small amount of CSS is often used to style the button and position it on the screen, making it more noticeable and user-friendly.

4. Initiating an Email

You can prompt the user’s default email client to open automatically with a predefined recipient address. Using the mailto: scheme within the href attribute makes setting up a ‘Contact Us’ button entirely effortless. for example:

<a href="mailto:support@example.com">email</a>

email

When creating an email link with the mailto: protocol, you can prefill several fields to make it easier for users to compose a message.

cc
Adds one or more recipients who will receive a visible copy of the email.
bcc
Adds one or more recipients who receive a hidden copy that other recipients cannot see.
subject
Sets the default subject line of the email.
body
Inserts predefined text into the message body.

To use these fields effectively, it is also important to understand a few special characters that appear in mailto: links:

?
Begins the list of email parameters. It is placed immediately after the email address.
&
Separates multiple parameters.

%20
Represents a space in a URL. Since spaces are not allowed directly in URLs, %20 is used instead.
%0D%0A
Represents a new line (line break) in the email body. This allows you to format longer messages with multiple lines.

Prefill email ;)

Note: Publishing an email address directly inside an <a href="mailto:...> link makes it easier for automated to discover and collect it. Some spammers and scammers use these bots to harvest email addresses from public websites. To reduce this risk, many developers generate email links dynamically with JavaScript or use other obfuscation techniques, making it more difficult for simple harvesting bots to detect the address. Keep in mind, however, that determined crawlers may still be able to extract email addresses, so JavaScript should be viewed as a deterrent rather than a complete security solution.

5. Triggering a Phone Call

Particularly useful for mobile users, the tel: scheme allows visitors to ring a specified telephone number directly. With a single tap, the device’s dialler is activated, making it an excellent feature for business websites.

For example

6. Downloading Files

If you want to offer a document, such as a PDF or an image, for the user to save locally, you can easily do so. Simply include the download attribute alongside the file’s path in the href, which instructs the browser to download the file rather than navigating to it.

for local file:


        <a href = "Local url or directory/file.name"> Download Link </a>


Download


Keep in mind that the "download" attribute is generally intended for files hosted on the same origin as your website. It cannot reliably force the download of files from external websites using HTML alone. Most browsers ignore the "download" attribute for cross-origin resources unless the remote server explicitly allows it through the appropriate HTTP headers. If you need to provide downloadable files from another domain, the server hosting the file must be configured to support downloads, or you must serve the file through your own backend or another trusted source under your control.
