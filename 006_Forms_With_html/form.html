1.Forms and Backend

To generate forms in HTML, you should use the <form> tag. In this section, we start with backend options, and in the next sections we will talk about frontend settings. Here you have six attributes: action, method, enctype, autocomplete, novalidate and target.

1.1.action

The action attribute gives you two options: it can be empty (#) or point to a server-side script.

1.2.method

You can choose between three different options for three different purposes. The first one is GET. Please do not use it for private data like login information and similar cases, because it will place the data in the URL and display it in the search bar. The second one is POST. You can use it for sensitive data, as it hides the data and sends it securely. The third one is dialog. It saves the data locally and does not send it to the server. So, sometimes you may want to handle data with JavaScript and only need some input from the user. In that case, I recommend using dialog.

Please be aware that POST only hides the data from the URL; it does not encrypt or secure it. This is one of the key reasons why we should use HTTPS.

1.3.enctype

Do you want your users to upload pictures, videos or other files? If so, you must set enctype="multipart/form-data"; otherwise, the form will not send anything to your server.

1.4.autocomplete

Have you ever seen some websites in registration forms or search bars recommend your previous input, email, name or other information? They actually use the autocomplete attribut

The autocomplete attribute can be set to "on" (default) or "off". You can also specify a particular value like "username", "email","current-password", "country", "postal-code" to help the browser suggest the correct type of data for each field.

1.5.novalidate

novalidate: Are you a master at JavaScript? Do you want to apply your own rules instead of the browser's default validation? Just add the novalidate attribute, and the validation process will be handled by your custom rules.

1.6.target

Do you remember the target attribute from the hyperlink practice? You haven't seen it? You can read about it here, but for a brief review: you can decide whether, after the user presses the submit button, the result should be displayed in the current page or in a new tab. If the user presses 'login', it is logical for the page to open in the current tab. However, if they press 'send comment', you might want to show a message like "Thank you for your report. We will review it shortly."



2.Label and Input

2.1.input

As you may have noticed, we haven't provided any examples for the backend section. Do you know why? It's because without front-end settings, the <form> tag doesn't display anything on its own. From this point onward, we are going to cover front-end settings using pure HTML. The first and most important element that makes the <form> tag visible is input.

for example: 


                    <form action="#" method="dialog" target="_blank" autocomplete="on" >
                        <input type="text">
                    </form>

As you saw in the previous example, the <input> element includes several attributes. But is that all? And what exactly does type="text" mean? Check out the following list and examples to learn more about the different attributes of the <input> element and how they work.

1.type
Specifies what type of data the user is expected to enter into the form field.

                            <form action="#" method="dialog">
                                <label for="test-01">password</label>
                                <input type="password" id="test-01">
                            </form>

2.id
The id attribute assigns a unique identifier to an <input> element. HTML, CSS, and JavaScript can use this identifier to reference the element. for example: . 

                            <style>
                                #test-id-name{background-color: orangered;}
                                #test-id-age{background-color: orange;}
                            </style>                            

                            <form action="#" method="dialog">
                                <label for="test-id-name">Name: </label>
                                <input type="text" id="test-id-name">
                            
                                <label for="test-id-age">Age</label>
                                <input type="number" type="test-id-age">
                            </form>

As you saw in the example above, we used the id attribute to reference the input elements from both HTML and CSS.
3.name
While id is commonly used to identify an element on the front end, the name attribute is used by backend languages such as PHP to identify the submitted form data.
4.value
The value attribute sets the initial value of an input field. It is especially useful when users are editing existing information, such as updating their profile. 

                            <form action="#" method="dialog" >
                                <label for="id-value">E-mail: </label>
                                <input type="email" id="id-value" value="example@email.com" >
                            </form>
5.placeholder
The placeholder attribute is used to display a hint or an example inside an input field. It's especially helpful when users need guidance on the expected format. For example, when entering a phone number, seeing a placeholder like +98... makes it immediately clear which format should be used. 

                            <form action="#" method="dialog">
                                <label for="id-placeholder">Phone Numbre: </label>
                                <input type="tel" id="id-placeholder" placeholder="+15555550199">
                            </form>
6.autofocus
Let me explain it with an example. Have you ever noticed that when you open Google, the cursor is automatically placed in the search box? That's exactly what the autofocus attribute does.
So, where is it useful? Personally, I like to place the cursor in the email field as soon as the sign-up page loads, allowing users to start typing immediately without making an extra click. 

                            <form action="#" method="dialog">
                                <label for="id-autofocus">Search: </label>
                                <input type="search" id="id-autofocus" autofocus>
                            </form>

In this example, to avoid the page jumping to the middle of the content caused by the autofocus attribute, we utilized JavaScript for simulation. If you implement the code you see above, everything will be performed automatically.
7.disabled
The disabled attribute disables an input field, preventing users from clicking on it, typing into it, or submitting its value with the form. However, it does not hide the element. The input remains visible, but users cannot interact with it.
When is it useful? Have you ever visited a website where you must first accept the terms and conditions before the other buttons or input fields become available? That's a common use case for the disabled attribute. In real-world applications, this behavior is usually controlled with a bit of JavaScript, which enables or disables form elements based on the user's actions. 

                            <form action="#" method="dialog">
                                <label for="id-disabled-user">User Name: </label>
                                <input type="text" id="id-disabled-user" disabled>

                                <label for="id-disabled-email">E-mail: </label>
                                <input type="email" id="id-disabled-email" disabled>

                                <label>
                                    <input type="checkbox" onchange=
                                    "document.getElementById('id-disabled-user').disabled = !this.checked;
                                    document.getElementById('id-disabled-email').disabled = !this.checked;
                                    ">
                                    I accept rules and conditions.
                                </label>
8.readonly
The readonly attribute allows users to view and copy the text, but they cannot modify its value. 

                            <form action="#" method="dialog">
                                <label for="id-read-only">Your Tracking Code</label>
                                <input type="number" id="id-read-only" value="123456789" readonly>
                            </form>
9.autocomplete
The autocomplete attribute in HTML <form> or <input> elements specifies whether the browser should automatically complete form fields based on the user's previous inputs.
It accepts values like on (enabled), off (disabled), or specific types such as name, email, address, etc., improving user experience by making form filling faster and more accurate. 

                            <form action="#" method="dialog">
                                <label for="id-autocomplete">Search bar: </label>
                                <input type="search" id="id-autocomplete" autocomplete="on" placeholder="type to find ...">
                            </form>
10.required
If the field is empty, the browser prevents submission and shows a validation message, ensuring essential data is provided by the user. 

                            <form action="#" method="dialog">
                                <label for="id-required">emali: </label>
                                <input type="email" id="id-required" required>

                                <label for="id-required-pass">password: </label>
                                <input type="password" id="id-required-pass" required>

                                <label for="id-required-check">accept rules and conditions</label>
                                <input type="checkbox" required>

                                <button type="submit">login</button>
                            </form>
11.maxlength & minlength
maxlength limits the maximum characters allowed, while minlength sets the minimum characters required in form inputs. 

                            <form action="#" method="dialog">
                                <label for="id-max-min">Choose a passcode</label>
                                <input type="password" name="We dont have backend" id="id-max-min"
                                        minlength="8" maxlength="40" required placeholder="like: co^v*lM09k..."
                                >

                                <button type="submit">
                                    Create
                                </button>
                            </form>
12.max & min
max sets the maximum value (for numbers, dates, etc.), while min defines the minimum acceptable value in form inputs. 

                            <form action="#" method="dialog">
                                <label for="id-min-max">Age: </label>
                                <input type="number" min="18" max="100">
                                <button type="submit">Send</button>
                            </form>
13.pattern
pattern attribute uses a regular expression (regex) to validate the format of user input (e.g., email, phone, or custom rules) in form fields. 

                            <form action="#" method="dialog">
                                <label for="id-pattern">Enter New E-mail: </label>
                                <input 
                                    type="email" title="Wrong email pattern" pattern="(\w+[\.\-_]*\w+)+@\w+(\.\w+){1,3}"
                                    required autocomplete="off" placeholder="email@example.com"
                                    id="id-pattern" name="nothing"
                                    >
                                <button type="submit">Sign in</button>
                            </form>
14.accept
The accept attribute lets you specify which types of files users are allowed to select. Have you ever noticed that when you're uploading a photo on a website, your folder may contain many different file types, but only the images are shown? That's exactly what the accept attribute does—it filters the file picker to display only the allowed file types.
15.multiple
The multiple attribute allows users to select multiple files at the same time or enter multiple email addresses in a single input field. 

                            <form action="#" method="dialog">
                                <label for="id-upload">Choose an Avatar: </label>
                                <input type="file" id="id-upload" name="upload"
                                        accept="image/*" multiple
                                >
                            </form>
16.step
The step attribute defines the interval between valid values for an input field. For example, imagine you're selling a product that can only be purchased in whole numbers. In that case, you wouldn't want customers to order 1.5 items. By setting step="1", only whole numbers are allowed. On the other hand, if you're selling a product by weight, you can use a smaller step value, such as 0.1 or 0.5, to allow decimal quantities.

17.inputmode
The inputmode attribute tells the browser which on-screen keyboard is most appropriate for the input field, especially on mobile devices. For example, when you tap an email field, the keyboard often includes the @ symbol for easier typing. Likewise, when entering a verification code, a numeric keyboard is usually displayed. That's exactly what the inputmode attribute is designed to do. 

                             <form action="#" method="dialog" >
                                 <label for="id-inputmode">Enter your phone numbre: </label>
                                 <input type="tel" name="inputmode" id="id-inputmode"
                                        required inputmode="tel"
                                >
                            </form>
18.list
The list attribute is used together with the <datalist> and <option> elements to provide a list of suggested values for an input field. You've probably used this feature before. For example, when filling out a registration form, you may start typing the name of your country and see a list of matching suggestions appear. That's a common use case for list and <datalist>. Look at here.
19.form
If your page layout requires an input field to be placed outside its main <form> element, the form attribute lets you associate it with that form. Simply assign an id to the <form> element, then set the form attribute of the external <input> to the same value. This links the input to the form, even though it is located elsewhere in the document. 

                            <form action="#" method="dialog" id="external">
                                <label for="id-from">Choose your free time</label>
                                <input type="time" id="id-form" name="form" required
                                    min="08:45" max="20:30"
                                >
                                <button type="submit">Search</button>
                        </form>

                        <label for="">Date: </label>
                        <input type="date"  required min="2026-08-26"
                                max="2026-11-29"    form="external"
                                >

2.2.label

As you may have noticed in the example above you can use the <label> element to associate descriptive text with each form control. This improves both usability and accessibility, as clicking on the label will focus the corresponding input field.

Remember to place the <label> element before the <input> element for better visual appearance in English texts. This label element has an attribute called for="Id name". In this part, you should use the same value that you assigned to the id attribute in the <input> element. 


                    <form action="#" method="dialog" autocomplete="on">
                        <label for="user-email">E-mail</label>
                        <input type="email" id="user-email" autocomplete="email">
                    </form>

Radio Buttons

Radio buttons are one of the form elements in HTML that present multiple options to the user, allowing them to select only one of them. They may seem similar to checkboxes in terms of usage, but the main difference is that checkboxes allow multiple selections at the same time, whereas radio buttons allow only one selection within a group.To create a radio button, you should use type="radio" in the input tag. An important point is that:

  • The name value must be the same for all options in the same group so that the user cannot select more than one option from that group.
  • The value attribute specifies what data will be submitted with the form when that option is selected.

Radio buttons are commonly used for choices such as gender, marital status, and other single-selection options.See the example below. 


                <form action="#" method="dialog">
                    <label for="id-gender-m">Male</label>
                    <input type="radio" name="gender" id="id-gender-m" value="male" required>

                    <label for="id-gender-f">Female</label>
                    <input type="radio" name="gender" id="id-gender-f" value="female">

                    <button type="submit">send</button>

4.Checkboxes

Checkboxes are form elements that allow the user to make multiple selections simultaneously. Unlike radio buttons, which restrict the user to choosing only one option, checkboxes give the user the freedom to select as many as they like. They are typically used for selecting skills, interests, or accepting terms of service. Keep in mind that for this, you should use type="checkbox".

Each checkbox acts independently. If you want to handle a set of checkboxes as a group in the backend, you usually use a similar name (often with [] in many backend languages to receive them as an array). value="" is, as before, the value sent to the server when that specific box is checked. See the example below. 


                <form action="#" method="dialog">
                    <label for="id-skill-html">html</label>
                    <input type="checkbox" name="skill[]" id="id-skill-html" value="html">

                    <label for="id-skill-css">css</label>
                    <input type="checkbox" name="skill[]" id="id-skill-css" value="css">

                    <label for="id-skill-JavaScript">JavaScript</label>
                    <input type="checkbox" name="skill[]" id="id-skill-JavaScript" value="JS">

                    <label for="id-skill-SQL">SQL</label>
                    <input type="checkbox" name="skill[]" id="id-skill-SQL" value="SQL">

                    <label for="id-skill-php">php</label>
                    <input type="checkbox" name="skill[]" id="id-skill" value="php">

                    <button type="submit">Send</button>
                </form>

5.Dropdown Lists

Dropdown menus are perfect when you have many options and don’t want to clutter the page with long rows of radio buttons or checkboxes. They allow the user to select one (or sometimes multiple) options from a collapsible list. They are commonly used for selecting cities, years of birth, or product categories. Keep in mind that to create this list, you use two main tags:

  • The <select> tag, which acts as the main container (and where you define the name attribute).
  • The <option> tag, which represents each item within the list.

value="" is placed inside each <option> tag and determines what value is sent to the backend if that option is selected.See the example below.

Example No.1: 


                <form action="#" method="dialog">
                    <label for="id-color">Select your favorit color</label>

                </form>

6.Comment Section

When you want the user to be able to write long text (like comments, bio, or messages), a standard <input> tag (which is only one line) won’t work. Here, we use the <textarea> tag. This tag gives the user enough space to write several lines of text. Keep in mind that:

  • Unlike <input>, the <textarea> tag has a closing tag (</textarea>).
  • To set the size of the typing area, we use the rows (number of lines) and cols (number of columns) attributes.
  • The name attribute is essential here as well, so the data can be sent to the server. 

        <form action="#" method="dialog">
            <label for="id-comment">Comment </label>
            <br>
            <textarea name="comment" id="id-comment" 
                cols="30" rows="6" maxlength="500"
                minlength="5" placeholder="type between 5 to 500 characters"
            ></textarea>
            <br>
            <button type="submit">Send</button>
        </form>
