1.Forms and Backend

To generate forms in HTML, you should use the <form> tag. In this section, we start with backend options, and in the next sections we will talk about frontend settings. Here you have six attributes: action, method, enctype, autocomplete, novalidate and target.

1.1.action

The action attribute gives you two options: it can be empty (#) or point to a server-side script.

1.2.method

You can choose between three different options for three different purposes. The first one is GET. Please do not use it for private data like login information and similar cases, because it will place the data in the URL and display it in the search bar. The second one is POST. You can use it for sensitive data, as it hides the data and sends it securely. The third one is dialog. It saves the data locally and does not send it to the server. So, sometimes you may want to handle data with JavaScript and only need some input from the user. In that case, I recommend using dialog.

Please be aware that POST only hides the data from the URL; it does not encrypt or secure it. This is one of the key reasons why we should use HTTPS.

1.3.enctype

Do you want your users to upload pictures, videos or other files? If so, you must set enctype="multipart/form-data"; otherwise, the form will not send anything to your server.

1.4.autocomplete

Have you ever seen some websites in registration forms or search bars recommend your previous input, email, name or other information? They actually use the autocomplete attribut

The autocomplete attribute can be set to "on" (default) or "off". You can also specify a particular value like "username", "email","current-password", "country", "postal-code" to help the browser suggest the correct type of data for each field.

1.5.novalidate

novalidate: Are you a master at JavaScript? Do you want to apply your own rules instead of the browser's default validation? Just add the novalidate attribute, and the validation process will be handled by your custom rules.

1.6.target

Do you remember the target attribute from the hyperlink practice? You haven't seen it? You can read about it here, but for a brief review: you can decide whether, after the user presses the submit button, the result should be displayed in the current page or in a new tab. If the user presses 'login', it is logical for the page to open in the current tab. However, if they press 'send comment', you might want to show a message like "Thank you for your report. We will review it shortly."



2.Label and Input

2.1.input

As you may have noticed, we haven't provided any examples for the backend section. Do you know why? It's because without front-end settings, the <form> tag doesn't display anything on its own. From this point onward, we are going to cover front-end settings using pure HTML. The first and most important element that makes the <form> tag visible is input.

for example: 


                    <form action="#" method="dialog" target="_blank" autocomplete="on" >
                        <input type="text">
                    </form>

As you saw in the previous example, the <input> element includes several attributes. But is that all? And what exactly does type="text" mean? Check out the following list and examples to learn more about the different attributes of the <input> element and how they work.

type
Specifies what type of data the user is expected to enter into the form field.
id
The id attribute assigns a unique identifier to an <input> element. HTML, CSS, and JavaScript can use this identifier to reference the element.
name
While id is commonly used to identify an element on the front end, the name attribute is used by backend languages such as PHP to identify the submitted form data.
value
The value attribute sets the initial value of an input field. It is especially useful when users are editing existing information, such as updating their profile.
placeholder
The placeholder attribute is used to display a hint or an example inside an input field. It's especially helpful when users need guidance on the expected format. For example, when entering a phone number, seeing a placeholder like +98... makes it immediately clear which format should be used.
autofocus
Let me explain it with an example. Have you ever noticed that when you open Google, the cursor is automatically placed in the search box? That's exactly what the autofocus attribute does.
So, where is it useful? Personally, I like to place the cursor in the email field as soon as the sign-up page loads, allowing users to start typing immediately without making an extra click.

2.2.label

As you may have noticed in the example above, the form feels somewhat unclear because there is no indication of what the user is expected to enter into the input field. To address this, you can use the <label> element to associate descriptive text with each form control. This improves both usability and accessibility, as clicking on the label will focus the corresponding input field.

Remember to place the <label> element before the <input> element for better visual appearance in English texts. This label element has an attribute called for="Id name". In this part, you should use the same value that you assigned to the id attribute in the <input> element. 


                    <form action="#" method="dialog" autocomplete="on">
                        <label for="user-email">E-mail</label>
                        <input type="email" id="user-email" autocomplete="email">
                    </form>
