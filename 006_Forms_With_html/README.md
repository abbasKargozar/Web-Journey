# HTML Forms Practice

This project is the final HTML practice in my **Web Journey** series. It focuses on HTML forms and covers both the theory and practical usage of the most common form elements and attributes.

Unlike a simple collection of examples, this page is structured as a small reference guide where each concept is explained, demonstrated with code, and followed by a working example.

## Topics Covered

* `<form>` element
* Backend-related attributes

  * `action`
  * `method`
  * `enctype`
  * `autocomplete`
  * `novalidate`
  * `target`
* `<input>` element and its most common attributes
* `<label>`
* Radio buttons
* Checkboxes
* Dropdown lists (`<select>` & `<option>`)
* `<textarea>`
* `<fieldset>` and `<legend>`
* Practical examples combining multiple form elements into complete forms

## Project Goal

The purpose of this practice was not only to learn HTML forms, but also to understand when and why each element or attribute should be used in real-world applications.

Since this project is hosted with GitHub Pages and has no backend, all forms use placeholder values such as `action="#"` or `method="dialog"` where appropriate. The primary goal is to demonstrate HTML behavior rather than server-side processing.
