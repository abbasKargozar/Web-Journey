How to Manipulate Multimedia in HTML

To work with multimedia in HTML, you only need four elements: 


                <img>
                <video>
                <audio>
Simply provide the path to your file, and the browser will display it. While CSS and JavaScript can improve the appearance, performance, and functionality of multimedia, HTML itself also offers a variety of useful attributes that let you control how these elements behave. In the following sections, we'll look at the most important attributes and the best practices you should follow when using each of these multimedia elements.

1.Displaying Images

To do this in HTML, we have two tags at our disposal: <img> and <picture>. To use these tags properly, you must be familiar with the following attributes:

  • src
  • height & width
  • alt
  • loading
  • type
  • decoding="async"
  • srcset

Simple photo

The most simple way to show a picture is : 


                <img src="your/picture/path or  url" >

As you see, the picture above this text isn't pleasant. I'm not referring to the absence of CSS. You may notice the enormous size of the picture. To avoid this, you should've used the "width" and height "attributes". These attributes also prevent CLS and are good for SEO. 


            <img src="./photo/sample_photo_01.png" width="650px" height="400px">

Other things that help your SEO and help blind people are the "alt" attribute. When your photo isn't available, it shows the alt text that you've written. And page scanners read the alt text instead of the photo's name, which might be annoying for blind people. 


            <img src="./photo/Not_Exist.png" width="650px" height="400px" alt="desktop photo">
        


            <img src="./photo/sample_photo_02.png" width="650px" height="400px" alt="desktop photo">

Lazy Loading

Do you want your page to load faster? One of the tricks related to this topic is using the "loading" attribute. By setting it to "loading=lazy", your page will load faster. But be aware: you should only use it for images that are at the bottom of the page and not yet visible to the user when the page first loads. 


            <img src="./photo/sample_photo_02.png" width="650px" height="400px" alt="desktop photo">



Decoding

Another answer to this question is using decoding="async". By doing this, you're telling the browser to interpret all of the code except this image, and after everything else, come back and interpret this one. This technique can be used for any image, but it's especially beneficial for images that are not in the initial viewport. 


            <img src="./photo/sample_photo_02.png" width="650px" height="400px" alt="desktop photo" decoding="async">



Picture Menu

How can upload image responsivly? for this matter you can use to helpful attributes. use "srset" to give your difrent options to browser. use "sizes" (max-width=) 100vw 50vw

srcset
The srcset attribute provides the browser with a list of available image files and their exact pixel widths, allowing it to automatically select and download the most optimal image for the user’s device.
sizes
The sizes attribute specifies how much screen space an image will occupy at various viewport widths, enabling the browser to choose the most appropriate image from the srcset before the CSS has loaded.

for example: 


                <img src="./photo/sample_photo_01.webp" 
                 srcset="./photo/small_sample_photo_01.jpg 400w, ./photo/medium_sample_photo_01.jpg 800w"
                 sizes="(max-width:600px) 100vw, 50vw"
                 alt="code photo"
            >

Here, sizes="(max-width:600px) 100vw, 50vw" means: 100vw sets the picture width to 100% if the screen is less than max-width (600px), and 50vw sets the picture width to 50% if the screen is greater than max-width (600px).

Video and SEO

If you want your page to load faster, you can use the preload attribute. It tells the browser to first interpret the page and then come back for the video files. By using the preload attribute, you have two options:

preload="none"
it waits for the user to press the play button, and then the browser looks for metadata and other information.
preload="metadata"
with this, the user can see the video time and video size, and it is the best option to choose.

To add subtitles or captions to videos (and sometimes audio files), the <track> tag is used inside the <video> element. The standard and common format for subtitle files on the web is WebVTT (with the .vtt extension). The <track> tag must always be placed inside the <video> or <audio> element, and after the <source> tags.

The most important attributes are:

src
For example: path/to/subtitle_file
kind
Defines the type of text track. Common values include subtitles, captions, chapters
srclang
Specifies the language of the subtitle file using a language code
label
Specifies the language of the subtitle file using a language code
default
ndicates that this text track should be enabled by default when the video loads. Only one <track> element can have the default attribute.
