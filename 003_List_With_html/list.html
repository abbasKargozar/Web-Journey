Create diffrent types of list in html

There are three different types of lists in HTML.

Unordered Lists

You need only two tag to create this type of list. One <ul> </ul>for creation of the base of the list and <li> </li> for example: 


            <ul>
                <li> content </li>
                <li> content </li>
                <li> content </li>
            </ul>

This is an Unordered list

CSS offers various list-style-type values, which we will introduce in the future. Without CSS, however, we are limited to the default disc value, which appears as a solid black circle

Ordered Lists

"Ordered lists are like unordered lists, but they order items using numbers, letters, or Roman numerals. You can also pick where to start counting and reverse the order if you want." 


            <ol>
                <li> content </li>
                <li> content </li>
                <li> content </li>
            </ol>
  1. type="1"
  2. type="a"
  3. type="A"
  4. type="i"
  5. type="I"

Note: You can reverse the order with 'reversed' attribute

Definition Lists

Definition lists are different from ordered and unordered lists. Instead of just listing items, they consist of term‑and‑description pairs. Each term is defined using the <dt> tag, and its corresponding description is provided with the <dd> tag. This structure makes definition lists ideal for glossaries, metadata, or any content that requires a term‑to‑definition relationship. 


            <dl>
                <dt> title </dt>
                <dd> description </dd>

                 <dt> title </dt>
                 <dd> description </dd>

                 <dt> title </dt>
                 <dd> description </dd>
            
            </dl>
Unordered lists
Displays list items with bullet points, used when the order of items doesn't matter.
Ordered Lists
Displays list items in a numbered or lettered sequence, used when the order matters.
Definition lists
Groups terms with their descriptions, used for glossaries, metadata, or definition pairs.

Nested List

"Lists can be nested inside one another to create hierarchical structures. You can place an unordered list inside an ordered list, or vice versa, and even mix different list types. This is particularly useful for creating outlines, sitemaps, multi‑level menus, or any content that requires multiple levels of indentation to show relationships between items."

This nesting capability is widely used in modern web development to build complex navigation systems, including dropdown menus, mega menus, and collapsible sidebars. By combining nested lists with CSS, you can create interactive and visually appealing navigation structures. For instance, a main list item can contain a sublist that appears only when hovered or clicked, forming the foundation of most website menus. These advanced styling techniques and interactions will be covered in greater detail in the CSS section.
