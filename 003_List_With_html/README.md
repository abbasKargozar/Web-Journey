# HTML Review – 003: Lists

## 📖 About

This project is the third exercise in my **HTML Review Series**.

The purpose of this exercise is to review every major type of HTML list and understand when and why each one should be used. Rather than focusing on visual design, this project emphasizes writing clean HTML and understanding the semantic purpose of each list element.

Styling is intentionally kept to a minimum because CSS will be reviewed in a separate section later.

---

## 🎯 Objectives

In this exercise I practiced:

- Unordered Lists (`<ul>`)
- Ordered Lists (`<ol>`)
- Definition Lists (`<dl>`)
- List Items (`<li>`)
- Definition Terms (`<dt>`)
- Definition Descriptions (`<dd>`)
- Nested Lists
- Ordered List Attributes

---

## 📚 What This Page Demonstrates

### 1. Unordered Lists

- Basic unordered lists
- Default list markers
- Introduction to `list-style-type`

### 2. Ordered Lists

Demonstrates different numbering styles using the `type` attribute:

- Numbers
- Uppercase letters
- Lowercase letters
- Uppercase Roman numerals
- Lowercase Roman numerals

It also introduces the `reversed` attribute.

### 3. Definition Lists

Shows how to associate terms with their descriptions using:

- `<dl>`
- `<dt>`
- `<dd>`

### 4. Nested Lists

Demonstrates how different list types can be combined to create hierarchical structures commonly used for:

- Website navigation
- Documentation
- Multi-level menus
- Content outlines

---

## 💡 Learning Notes

During this exercise I learned that:

- Every list serves a different purpose.
- Ordered lists should be used only when sequence matters.
- Definition lists are ideal for term-description relationships.
- Lists can be nested to create hierarchical structures.
- HTML provides the structure, while CSS controls the appearance of list markers.
